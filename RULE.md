### url 설정시 규칙 
1. 마지막에 / 포함하지 않는다.

2. _(underbar) 대신 -(dash)를 사용한다.

3.소문자를 사용한다.

4.행위(method)는 URL에 포함하지 않는다.

5.컨트롤 자원을 의미하는 URL 예외적으로 동사를 허용한다.



### git 사용법
1. 회의록은 issue에 작성한다.
2. todo, -ing, done 등 진행상황은 해당 repository의 project에 작성한다.
